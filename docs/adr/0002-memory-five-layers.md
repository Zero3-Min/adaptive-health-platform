# ADR-0002: 五层记忆架构

- 状态：Accepted
- 日期：2026-07-09

## 背景

"Memory First" 是本系统的核心原则：个性化与持续学习的能力上限由记忆系统决定，而非由 Agent 提示词决定。单一的"对话历史 + 向量库"无法区分事实、事件、结论、策略与元认知，导致检索噪声大、演进不可控。

## 决策

将记忆划分为五层，各层有独立的写入路径、生命周期与置信语义：

| 层 | 内容 | 写入方 | 变化频率 | 示例 |
|---|---|---|---|---|
| 1. Profile | 用户静态/半静态事实：目标、约束、偏好、健康基线 | 用户输入 + 确认后的推断 | 低 | "膝伤史，避免深蹲"；"目标：减脂 5kg" |
| 2. Daily Timeline | 原始事件流，append-only | workflow 摄入管道 | 每日多次 | "07:30 晨跑 5km"；"睡眠 6.2h" |
| 3. Insights | 从 Timeline 提炼的模式与结论，带证据引用 | Reflection Agent | 每周期 | "连续熬夜后次日训练完成率降 40%" |
| 4. Strategy | 当前生效的干预策略与计划 | Coach Agent + 用户确认 | 按需 | "本月采用 3 练 1 休；晚 10 点睡眠提醒" |
| 5. Evolution | 关于系统自身的记忆：哪些策略/规则对该用户有效 | evaluation + evolution 模块 | 慢 | "对该用户，温和提醒比数据警告依从性高" |

### 关键规则

1. **单向提炼**：高层记忆必须能追溯到低层证据（Insight 引用 Timeline 事件，Strategy 引用 Insight）。禁止无证据的高层写入。
2. **Timeline 不可变**：第 2 层 append-only，修正以补充事件表达，保证回放（evolution/harness）可重现。
3. **分层检索**：Agent 上下文组装按层拉取——Profile 全量、Timeline 按时间窗、Insights/Strategy 按向量相关性（pgvector）、Evolution 按规则匹配。
4. **各层独立可测**：每层是 `core/memory` 中独立的读写接口，可单独 mock 与评估。

## 备选与否决

- **单一向量库记忆**：检索无结构，事实与猜测混杂，无法支撑"可解释"要求。
- **对话历史即记忆**：Agent Second 原则下系统大量行为不经对话产生，对话历史覆盖不了设备数据与工作流产物。
- **三层（事实/事件/总结）**：缺少 Strategy 与 Evolution 层，策略变更与系统自我认知无处落地，"Evolution First" 无法实现。

## 后果

- 写入路径变多，需要 `core/memory` 提供严格的分层 API 并在 evaluation 中监控各层质量。
- Reflection Agent 的产出质量直接决定第 3 层价值，需要最早建立评估基准。
- 换取的能力：任意建议可解释（逐层引用链）、系统可回放、演进可度量。
